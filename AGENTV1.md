## TL;DR тАФ the only architectural rule

**Agents propose, deterministic code disposes.**

- Agents never call broker APIs directly. Every order routes through `packages/engine/risk` (Phase 1) тЖТ `packages/broker`.
- Agents never originate raw data fetches. They consume pre-computed feature dicts.
- LLM output is never the kill-switch. Risk vetoes are pure Python with named rules (`pdt_block`, `daily_drawdown_halt`, `max_position_pct_trim`, etc.).
- Strategy code generated by an LLM is sandbox-tested тЖТ versioned artifact тЖТ loaded at runtime. Never `eval()` LLM output in the live path.

If a feature request would put LLM output inside a risk decision or execution path, push back before coding.

## What's deliberately NOT built yet (don't surprise yourself)

- **Postgres is opt-in, not default.** `USE_POSTGRES=1` env routes through `PostgresStore`. Without the env the API uses `MockStore` (in-memory). The migration is applied via `make migrate`.
- **No LangGraph nodes beyond the 5 in `apps/agents/trading_agents/nodes/`.** Phase 2 adds Macro Analyst, splits Strategy Selector тЖФ Proposal Drafter, and adds a Reflection node.
- **`engine.risk` ships 9 rules with 21 tests** (11 unit + 4 backtester-gate integration + 6 reconciler). Risk Officer node is a thin adapter over `engine.risk.evaluate`; the backtester routes proposals through the same evaluator via `RiskGate`. Still deferred: `correlation_cap` (same-cluster position count) and `wash_sale_tracking` (US tax, informational). PLAN.md ┬з6.2 lists them explicitly.
- **Reconciler ships in MOCK mode only.** `engine.reconciler.Reconciler` writes `positions_snapshot` + flips `circuit_breaker_state` on a 30s loop. Default poller is `MockBrokerPoller` (synthetic state). `AlpacaBrokerPoller` exists but is not yet wired тАФ wait for paper-trading validation (PLAN.md ┬з11 Phase 4) before flipping it on.
- **`PostgresStore.get_account` now reads from `positions_snapshot`** (newest row, falls back to cold-boot fixture if no row yet). `MockStore` still serves its fixture тАФ no behavior change in the default path.
- **`MockRiskContextProvider` is still the default for the agent council** when `USE_POSTGRES` is unset. With `USE_POSTGRES=1`, the council automatically uses `PostgresRiskContextProvider`. Same env switch as `app.services.store`.
- **21 tests green.** `packages/engine/tests/test_risk.py` (11) + `test_backtester_risk.py` (4) + `test_reconciler.py` (6) = 21 passed in 5.78s. Run with `PYTHONPATH=packages/engine:packages/broker pytest packages/engine/tests/ -v`. Reconciler Postgres integration tests are scaffolded as `@pytestmark_postgres` stubs in `test_reconciler.py` тАФ opt-in via `RUN_POSTGRES_TESTS=1`.
- **No CI run.** Workflow file is written; lockfiles (`pnpm-lock.yaml`, `uv.lock`) haven't been committed. **The user has explicitly asked the next agent NOT to run a CI dry-run** тАФ they'll do it themselves later. Don't run `pnpm install` or `uv sync` and commit the lockfiles unprompted.
- **No auth, no biometric flow, no token encryption.** API is open in Phase 0. PostgresStore writes everything against a single fixture user (`00000000-0000-0000-0000-000000000001`). Phase 3 lands real auth.

When in doubt, treat undocumented work as not-done.

## Conventions

### Python (`apps/api`, `apps/agents`, `packages/broker`, `packages/engine`)
- Python 3.12. `from __future__ import annotations` at top of every file.
- `ruff` lint+format, `mypy --strict`. CI enforces.
- `Protocol` over ABC where duck-typing helps. Use ABC when you need MRO behavior.
- Every public function has a one-line docstring.
- Comments explain WHY only тАФ never WHAT (variable names handle that).
- Every risk rule must have a stable name string (`veto_rule: str`) for audit logs.

### TypeScript / React Native (`apps/mobile`, `packages/ui`, `packages/shared-types`)
- TypeScript strict mode. `tsc --noEmit` must pass.
- NativeWind class strings via the `cn()` helper in `packages/ui/src/utils.ts`.
- **All colors come from design tokens.** Never raw hex in components. The `palette[mode]` map in `packages/ui/src/tokens.ts` is the source of truth, mirrored into `apps/mobile/tailwind.config.js`.
- Every interactive element: `accessibilityLabel` required, 44pt minimum tap target.

### Design system enforcement (from DESIGN.md)
- **Approve button is `accent-primary`, NEVER `gain`/green.** Green is reserved for fills + positive P&L.
- Numerals use Inter with `font-variant-numeric: tabular-nums`. The `<PriceDisplay>` primitive enforces this.
- Drawdown circuit-breaker UI uses `danger` token with a persistent banner that requires explicit user acknowledgement.
- Both Light and Dark must be tested before any UI commit.

### Git hygiene
- Branch: `agent-v1` is the active line. Feature work: `agent-v1/<scope>`.
- Conventional Commits prefix: `feat:`, `fix:`, `chore:`, `docs:`, `refactor:`, `test:`.
- Never `--no-verify`, never `--force` on a shared branch, never `reset --hard` without explicit user instruction.
- Don't commit secrets. `.env` is gitignored тАФ use `.env.example` for templates.
